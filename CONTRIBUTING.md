# 貢献ガイドライン

このプロジェクトへの貢献を歓迎します！以下のガイドラインに従ってください。

## プルリクエストのプロセス

1. **Issue を作成**: 大きな変更の場合は、まず Issue を開いて議論してください
2. **Fork & Clone**: リポジトリをフォークし、ローカルにクローン
3. **ブランチ作成**: `feature/your-feature-name` または `fix/issue-number` のような名前で
4. **変更を実装**: コードを書き、テストを追加
5. **コミット**: 明確なコミットメッセージを書く
6. **プッシュ & PR**: 変更をプッシュし、プルリクエストを作成

## コーディング規約

### JavaScript/React
- ES6+ の機能を使用
- 関数コンポーネントとフックを優先
- わかりやすい変数名・関数名を使用
- コメントは日本語または英語で

### スタイリング
- Tailwind CSS のユーティリティクラスを使用
- インラインスタイルは最小限に
- レスポンシブデザインを考慮

## 貢献できる領域

### 🎮 ゲーム機能
- 新しいシナリオの追加
- 新しいキャラクターの追加
- ゲームメカニクスの改善
- UI/UX の向上

### 📊 研究機能
- データ収集機能の追加
- 分析ツールの統合
- Firebase などのバックエンド統合

### 🌍 国際化
- 英語版の作成
- 他言語への翻訳

### 📚 ドキュメント
- README の改善
- 使用方法の詳細化
- コメントの追加

### 🐛 バグ修正
- バグ報告
- バグ修正のプルリクエスト

## 倫理的ガイドライン

このプロジェクトは**教育目的**です。以下を守ってください：

✅ **推奨される貢献**:
- 教育効果を高める機能
- ユーザビリティの向上
- デブリーフィングの充実
- 批判的思考を促す要素

❌ **避けるべき変更**:
- 実際の悪用を助長する機能
- デブリーフィングの削除
- 倫理的警告の削除
- 誤情報そのものの再生産

## テスト

プルリクエストを送る前に：
- ゲームが正常に動作することを確認
- 異なるブラウザでテスト
- モバイルでも動作確認

## コミットメッセージ

わかりやすいコミットメッセージを：

```
feat: 新しいシナリオを追加（災害時デマ）
fix: 説得度メーターのバグを修正
docs: README に Firebase 統合の説明を追加
style: コードフォーマットを統一
refactor: 状態管理を改善
```

## 質問・提案

不明な点や提案がある場合は、遠慮なく Issue を開いてください！
